绘制图形

  • 自定义路径Path

    我们需要使用到Path类，该类可以使用方法添加不同的路径信息，Path有两类方法：

    第一类：直接描述路径

    第一组：添加子图形

    还可以细分为两组，添加子图形和画线（直线或曲线）

    添加圆addCircle(100f, 100f, 50f, Direction.CW)

    参数1、2用于确定圆心位置，参数3为半径，参数4为路径方向

    路径方向有两种：顺时针 (CW clockwise) 和逆时针 (CCW counter-clockwise) 。对于普通情况，这个参数填 CW 还是填 CCW 没有影响。它只是在*需要填充图形(Paint.Style 为 FILL 或 FILL_AND_STROKE*) ，并且图形出现自相交时，用于判断填充范围的

    添加矩形addRect(120f, 120f, 400f, 400f, Direction.CW)

    参数1、2为确定矩形左上角的点坐标，参数3、4为确定矩形右下角的点坐标，参数5同样为缠绕轮廓方向

    当然还有添加椭圆、圆角矩形等方法

    第二组：画线（直线或曲线）

    画直线lineTo(float x, float y) / rLineTo(float x, float y)

    画二次贝塞尔曲线quadTo(float x1, float y1, float x2, float y2) / rQuadTo(float dx1, float dy1, float dx2, float dy2)

    参数1、2为红色线标出的坐标点，参数3、4为绿色线标出的坐标点（不一定正确）

    当然还有三次贝塞尔曲线，也就是多加了一组点

    移动到目标位置，也相当于设置了目标起点moveTo(float x, float y) / rMoveTo(float x, float y)

    画弧形 arcTo(float left, float top, float right, float bottom, float startAngle, float sweepAngle, boolean forceMoveTo)

    末尾的标志表示是否生成移动过来的痕迹，true为不生成，false为生成

    简化版添加弧形addArc(float left, float top, float right, float bottom, float startAngle, float sweepAngle)默认forceMoveTo为true

    封闭图形close()也就是说首尾点相连接

    注意：不是所有的子图形都需要封闭，当需要填充图形，也就是Paint.Style为FILL或者FILL_AND_STROKE时，会自动封闭图形

    自动封闭

    第二类：辅助的设置或计算

    设置填充方式setFillType(FillType fillType)

    前面在说 dir 参数的时候提到， Path.setFillType(fillType) 是用来设置图形自相交时的填充算法的：

    方法中填入不同的 FillType 值，就会有不同的填充效果。FillType 的取值有四个：

    • EVEN_ODD
    • WINDING （默认值）
    • INVERSE_EVEN_ODD
    • INVERSE_WINDING

    其中后面的两个带有 INVERSE_ 前缀的，只是前两个的反色版本，所以只要把前两个，即 EVEN_ODD 和 WINDING，搞明白就可以了。

    EVEN_ODD 和 WINDING 的原理有点复杂，直接讲出来的话信息量太大，所以我先给一个简单粗暴版的总结，你感受一下： WINDING 是「全填充」，而 EVEN_ODD 是「交叉填充」：

    之所以叫「简单粗暴版」，是因为这些只是通常情形下的效果；而如果要准确了解它们在所有情况下的效果，就得先知道它们的原理，即它们的具体算法。

    EVEN_ODD 和 WINDING 的原理

    EVEN_ODD

    即 even-odd rule （奇偶原则）：对于平面中的任意一点，向任意方向射出一条射线，这条射线和图形相交的次数（相交才算，相切不算哦）如果是奇数，则这个点被认为在图形内部，是要被涂色的区域；如果是偶数，则这个点被认为在图形外部，是不被涂色的区域。还以左右相交的双圆为例：

    射线的方向无所谓，同一个点射向任何方向的射线，结果都是一样的，不信你可以试试。

    从上图可以看出，射线每穿过图形中的一条线，内外状态就发生一次切换，这就是为什么 EVEN_ODD 是一个「交叉填充」的模式。

    WINDING

    即 non-zero winding rule （非零环绕数原则）：首先，它需要你图形中的所有线条都是有绘制方向的：

    然后，同样是从平面中的点向任意方向射出一条射线，但计算规则不一样：以 0 为初始值，对于射线和图形的所有交点，遇到每个顺时针的交点（图形从射线的左边向右穿过）把结果加 1，遇到每个逆时针的交点（图形从射线的右边向左穿过）把结果减 1，最终把所有的交点都算上，得到的结果如果不是 0，则认为这个点在图形内部，是要被涂色的区域；如果是 0，则认为这个点在图形外部，是不被涂色的区域。

    和 EVEN_ODD 相同，射线的方向并不影响结果。

    所以，我前面的那个「简单粗暴」的总结，对于 WINDING 来说并不完全正确：如果你所有的图形都用相同的方向来绘制，那么 WINDING 确实是一个「全填充」的规则；但如果使用不同的方向来绘制图形，结果就不一样了。

    图形的方向：对于添加子图形类方法（如 Path.addCircle() Path.addRect()）的方向，由方法的 dir 参数来控制，这个在前面已经讲过了；而对于画线类的方法（如 Path.lineTo() Path.arcTo()）就更简单了，线的方向就是图形的方向。

    而 INVERSE_EVEN_ODD 和 INVERSE_WINDING ，只是把这两种效果进行反转而已，你懂了 EVEN_ODD 和 WINDING ，自然也就懂 INVERSE_EVEN_ODD 和 INVERSE_WINDING 了，我就不讲了。

    好，花了好长的篇幅来讲 drawPath(path) 和 Path，终于讲完了。同时， Canvas 对图形的绘制就也讲完了。图形简单时，使用 drawCircle() drawRect() 等方法来直接绘制；图形复杂时，使用 drawPath() 来绘制自定义图形。

增加颜色

增加颜色有三层处理

  • 基本颜色
  • ColorFilter
  • Xfermode

基本颜色-Paint第一类API

基本颜色中有下列的方法设置颜色

  1. setColor

  2. setARGB

  3. setShader（着色器）

    1. LinearGradient 线性渐变

    2. RadialGradient 辐射渐变

    3. SweepGradient 扫描渐变

    4. BitmapShader 图像填充着色

    5. ComposeShader 混合着色器

      需要设置图像的混合模式，可参见官方文档查看具体有哪些模式[PorterDuff.Mode](PorterDuff.Mode | Android Developers)

ColorFilter

为绘制设置颜色过滤，通过Paint.setColorFilter(ColorFilter filter)进行设置，而ColorFilter不可直接使用，有3个子类可使用

  1. LightingColorFilter

    主要原理为对RGB颜色进行乘、加，从而改变RGB值

  2. PorterDuffColorFilter

    类似于前面的ComposeShader 混合着色器，设置的颜色可以和图形进行一系列的操作

  3. ColorMatrixColorFilter

    （不太懂）使用一个 ColorMatrix 来对颜色进行处理。 ColorMatrix 这个类，内部是一个 4x5 的矩阵

Xfermode

该类只有一个子类PorterDuffXfermode，用于处理图层混合模式，参数同样的需要设置PorterDuff.Mode

效果-Paint第二类Api

效果类的 API ，指的就是抗锯齿、填充/轮廓、线条宽度等等这些。

设置抗锯齿

有两种方法进行设置

  1. setAntiAlias(boolean aa)
  2. Paint(Paint.ANTI_ALIAS_FLAG)

设置样式

  1. setStyle

设置线条形状

  1. setStrokeWidth(float width) 线条宽度
  2. setStrokeCap(Paint.Cap cap) 线头形状
  3. setStrokeJoin(Paint.Join join) 拐角形状
  4. setStrokeMiter(float miter) 拐角补偿

色彩优化

  1. setDither(boolean dither) 设置图形抖动

    指把图像从较高色彩深度（即可用的颜色数）向较低色彩深度的区域绘制时，在图像中有意地插入噪点，通过有规律地扰乱图像来让图像对于肉眼更加真实的做法。

  2. setFilterBitmap(boolean filter) 设置双线性过滤

增加图片轮廓

setPathEffect(PathEffect effect)可以对轮廓进行风格化

存在多种子类

注意：PathEffect部分情况不支持硬件加速，比如drawLines

最好提前关闭硬件加速

  1. CornerPathEffect 拐角变成圆角

  2. DiscretePathEffect 对线条进行随机偏离，让轮廓变乱

  3. DashPathEffect 设置虚线轮廓

  4. PathDashPathEffect 使用一个Path绘制虚线，也可以立即为虚线的每一小段都是一个Path进行绘制的

  5. SumPathEffect 组合效果，按照两种PathEffect组合绘制

  6. ComposePathEffect 组合绘制，但是存在先后关系，先对目标path使用第一个PathEffect，然后再使用第二个

设置阴影

setShadowLayer(float radius, float dx, float dy, int shadowColor)

可以在之后绘制的内容下面添加一层阴影

clearShadowLayer()清除阴影

注意，硬件加速下只支持文字的绘制，文字之外的绘制需要关闭硬件加速

如果 shadowColor 是半透明的，阴影的透明度就使用 shadowColor 自己的透明度；而如果 shadowColor 是不透明的，阴影的透明度就使用 paint 的透明度。

设置绘制层上方的附加效果

setMaskFilter(MaskFilter maskFilter)

  1. BlurMaskFilter 模糊效果
  2. EmbossMaskFilter 浮雕效果

获取绘制的Path

  1. getFillPath(Path src, Path dst) 获取Path的实际Path

  2. getTextPath(String text, int start, int end, float x, float y, Path path) / getTextPath(char[] text, int index, int count, float x, float y, Path path) 获取文字的Path

    对文字而言，底层绘制时也是将其转换为图形进行绘制的

这两个方法主要用于图像和文字的装饰效果的位置计算，比如自定义的下划线效果

drawText()相关

比如设置文字大小、间隔、效果等

绘制文字的方式

Canvas绘制文字的方法有三个

  1. drawText() 注意该方法通过坐标去绘制，坐标为文字左下角
  2. drawTextRun() 根据上下文来绘制文字（针对部分语言，如阿拉伯语）
  3. drawTextOnPath() 沿着一条Path来绘制文字

Paint对文字绘制的辅助

设置显示效果类

  1. setTextSize(float textSize) 设置字体大小

  2. setTypeface(Typeface typeface) 设置字体

  3. setFakeBoldText(boolean fakeBoldText) 是否使用伪粗体

  4. setStrikeThruText(boolean strikeThruText) 是否加删除线

  5. setUnderlineText(boolean underlineText) 是否加下划线

  6. setTextSkewX(float skewX) 设置文字横向错切角度。其实就是文字倾斜度的啦

  7. setTextScaleX(float scaleX) 设置文字横向放缩。也就是文字变胖变瘦

  8. setLetterSpacing(float letterSpacing) 设置字符间距。默认值是 0

  9. setFontFeatureSettings(String settings) 用 CSS 的 font-feature-settings 的方式来设置文字

  10. setTextAlign(Paint.Align align) 设置文字的对齐方式

  11. setTextLocale(Locale locale) / setTextLocales(LocaleList locales) 设置绘制所使用的 Locale

  12. setHinting(int mode) 设置是否启用字体的 hinting （字体微调）

  13. setElegantTextHeight(boolean elegant) 设置是否开启文字的 elegant height

    仅针对于部分语言的部分文字，如果文字高度较高，系统会开启压缩，调用该方法会恢复文字的高度

  14. setSubpixelText(boolean subpixelText) 是否开启次像素级的抗锯齿

  15. setLinearText(boolean linearText) 否启用线性文本标志

测量文字尺寸类

  1. float getFontSpacing() 获取推荐的行距

  2. FontMetircs getFontMetrics() 获取 Paint 的 FontMetrics

    FontMetrics 是个相对专业的工具类，它提供了几个文字排印方面的数值：ascent, descent, top, bottom, leading

  3. getTextBounds(String text, int start, int end, Rect bounds) 获取文字的显示范围

  4. float measureText(String text) 测量文字的宽度并返回

  5. getTextWidths(String text, float[] widths) 获取字符串中每个字符的宽度，并把结果填入参数 widths

  6. int breakText(String text, boolean measureForwards, float maxWidth, float[] measuredWidth) 测量文字宽度

    和 measureText() 的区别是， breakText() 是在给出宽度上限的前提下测量文字的宽度。如果文字的宽度超出了上限，那么在临近超限的位置截断文字

  7. 光标相关

    1. getRunAdvance(CharSequence text, int start, int end, int contextStart, int contextEnd, boolean isRtl, int offset) 对于一段文字，计算出某个字符处光标的 x 坐标

    2. getOffsetForAdvance(CharSequence text, int start, int end, int contextStart, int contextEnd, boolean isRtl, float advance) 给出一个位置的像素值，计算出文字中最接近这个位置的字符偏移量（即第几个字符最接近这个坐标）

      getOffsetForAdvance() 配合上 getRunAdvance() 一起使用，就可以实现「获取用户点击处的文字坐标」的需求。

      获取用户点击处的文字坐标：应用场景

      1. 文本编辑器和富文本编辑器

      • 光标定位：在用户点击文本时，将光标准确地定位到点击位置。
      • 选择文本：允许用户通过点击和拖动来选择文本区域，并对选中的文本执行操作（如复制、剪切、格式化等）。
      • 文本高亮：用户点击某个单词或短语时，动态地高亮显示该部分文本，以便进行编辑或查看详细信息。

      2. 注释和批注功能

      • 添加注释：在阅读应用、文档编辑器或学习软件中，用户可以点击文本的某个位置添加注释、批注或评论。
      • 显示批注：用户点击含有批注的文字，显示相关的批注内容或注释。

      3. 超链接和交互文本

      • 点击链接：在显示超链接的文本中，识别用户点击的位置，确定用户点击的是哪个链接，然后执行相应的跳转或操作。
      • 触发事件：用户点击某个特定单词或短语，触发相应的事件或功能，例如显示详细信息、播放音频、打开对话框等。

      4. 教育和学习应用

      • 单词解释：用户点击某个单词时，显示该单词的解释、翻译或发音。
      • 知识点标记：用户点击某个知识点或重要术语，显示详细的背景信息或相关的学习资料。

      5. 互动式广告

      • 精确点击统计：在广告文本中，记录用户点击的位置，分析用户对哪部分内容更感兴趣，以优化广告效果。
      • 广告互动：用户点击广告文本中的特定部分，触发相应的互动活动，如弹出优惠信息、跳转到商品页面等。

      6. 用户行为分析

      • 点击热图：收集用户点击的文本位置数据，生成点击热图，分析用户关注的内容，优化内容布局和设计。
      • 用户体验改进：通过分析用户点击位置的数据，改进界面设计和交互体验，使其更加符合用户习惯。

  8. hasGlyph(String string) 检查指定的字符串中是否是一个单独的字形 (glyph）。最简单的情况是，string 只有一个字母（比如 a）

    比如一个emoji就算一个单独的字形，两个字符”ab”就不算
