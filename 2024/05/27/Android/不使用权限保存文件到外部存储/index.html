不使用权限保存文件到外部存储

发表于2024-05-27|更新于2024-05-27|Android
|字数总计:1k|阅读时长:4分钟|阅读量:

需求：保存图片或者其他文件到共享存储中，比如DCIM、PICTURE这种文件夹中

我们可以使用MediaStore作用域存储完成，但是仅适用于Android10及更高的版本

  1. 首先构建一个ContentValue，需要填入MediaStore.MediaColumns.DISPLAY_NAME、MediaStore.MediaColumns.MIME_TYPE、MediaStore.MediaColumns.DATA，而最后这个DATA需要根据不用的版本做处理
  2. 使用ContentResolver的api将上面的内容插入到MediaStore.Images.Media.EXTERNAL_CONTENT_URI中，可以构建出一个uri对象
  3. 为uri写入数据，可以做版本适配，SDK29及其以上可以通过使用contentValue避免其他应用程序对该文件进行读写

为什么可以不需要权限就保存文件

从Android 10开始，引入了作用域存储机制（Scoped Storage）。在这种机制下，应用可以在不请求存储权限的情况下通过MediaStore API 向公共集合（如相册）中添加媒体文件。这是因为MediaStore为应用提供了一个安全的方式来访问和修改这些公共集合中的文件，不需要应用拥有对整个文件系统的访问权限。

总结： 这段代码利用了Android 10及以上版本的作用域存储特性，通过MediaStore API 安全地向公共集合中添加图片数据，不需要请求读写存储权限。而在低于Android 10的版本中，仍然需要手动处理文件路径和权限问题，但这段代码中没有处理权限请求，这意味着在实际应用中可能需要额外的权限请求逻辑来保证在低版本系统上的正常运行。
